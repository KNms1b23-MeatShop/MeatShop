# Cart & Checkout — Draft Notes (Issue #11 / #13)

Цей документ описує план реалізації логіки кошика та підготовки до оформлення замовлення у проєкті «М’ясна Лавка».
Мета — стабільна робота кошика: зміна кількості, правильний підрахунок суми, empty state, підготовка до checkout.

---

## Scope
### In scope (MVP)
- Зміна кількості товару кнопками `+` / `-` (мінімум 1)
- Видалення позиції з кошика
- Empty state (коли кошик порожній)
- Коректний підрахунок total (без “накопичення”)
- Оновлення лічильника кошика в header/footer (якщо використовується)

### Out of scope (поки що)
- Оплата онлайн
- Бекенд-замовлення
- Реальна авторизація/кабінет користувача

---

## Data model (логіка даних)
Кожна позиція в кошику:
- `id` — ідентифікатор товару
- `name` — назва
- `price` — ціна за 1 одиницю (грн/кг або грн/шт — як у даних)
- `qty` — кількість (мін. 1)

Зберігання (якщо застосовується):
- `localStorage` key: `cart` (масив позицій)

---

## UI behavior
- Кнопка `+` збільшує `qty` на 1
- Кнопка `-` зменшує `qty` на 1, але **не нижче 1**
- Кнопка “Видалити” прибирає позицію з кошика
- Якщо кошик порожній — показати повідомлення (empty state) + CTA “Перейти в магазин”
- Total показується як сума по всіх позиціях: `sum(price * qty)`

---

## Total calculation (важливо)
Total не має “накопичуватися”.
Правильний підхід: кожен раз при рендері/оновленні рахувати total з нуля.

Псевдокод:
```js
function calcTotal(items) {
  let total = 0;
  for (const item of items) total += item.price * item.qty;
  return total;
}
